Visual design & interaction improvements for the Audiobookshelf client

1

Mini Player — Richer, More Informative Bar

Medium

The current mini player is a single slim bar. Surface more info at a glance: chapter title, a tiny chapter-scoped progress ring around the cover art, and a waveform pulse to show live playback state. Add a swipe-down-to-dismiss gesture.

  • Thin progress bar at bottom of mini player for book-level progress

2

Library Grid — Richer Cover Overlays

Easy

Book covers in the grid currently show minimal state. Add a subtle bottom progress bar, a "finished" checkmark badge, a "new" badge for recently added items, and a "currently playing" animated border so users can find their active book instantly.

  • Progress bar height: 5–6px at very bottom edge of cover image
  • Glowing purple border + play icon for currently-playing book — easy to spot in large libraries
  • "Finished" green checkmark badge stays until user manually marks unread
  • Orange "NEW" pill badge (configurable: last 7 days)
3

Full Player — Intuitive Swipe Gesture Zones

Medium

The full-screen player has no swipe gestures for navigation. Add gesture zones: swipe left/right on the cover for previous/next chapter, swipe up from cover to reveal the chapter list, and swipe down to dismiss back to the mini player. Haptic feedback on chapter change.

  • Swipe up anywhere on player opens chapter list bottom sheet
  • Active chapter highlighted with accent color left border
  • Swipe down from anywhere dismisses player back to mini player
4

Search — Persistent Bar with Filter Chips

Easy

The search bar currently hides when scrolling. Make it sticky at the top and add horizontal filter chips below it (All / Unfinished / Downloaded / By Series / By Author). Chips update results instantly without navigating away.

  • Search bar pins to top — results list scrolls beneath it
  • Filter chips: All, Unfinished, Downloaded, In Series, By Author (horizontal scroll if more)
  • Active chip uses accent fill; inactive chips use subtle border only
  • Chips persist between sessions via ui_prefs.dart
5

Sleep Timer — Visual Countdown Ring

Easy

When a sleep timer is active, the user has no visual cue in the player. Draw a thin arc around the cover art that depletes as time runs out. Show minutes remaining in a small badge. Tapping it opens the timer editor directly.

  • Thin orange arc around cover art depletes over the timer duration
  • When <5 minutes remain the arc pulses to warn the user
  • Timer badge (top-right of cover) shows minutes remaining; tap to modify
  • No ring shown when timer is inactive — zero visual clutter
6

Book Detail — Timeline Bar + Related Books Row

Medium

Book detail pages show cover, description, and a play button. Add a chapter timeline bar (each chapter as a colored segment, current position marked) and a "More like this" horizontal carousel at the bottom for books in the same series or by the same author.

Book Detail (enhanced)
  • Chapter timeline: each segment proportional to chapter duration, completed chapters shown darker
  • White tick marker shows exact current playback position across all chapters
  • "More in Series" row auto-populates from series.dart model already in code
  • Tapping a series card opens that book's detail page
7

Stats Page — Bar Chart Listening History

Hard

The stats page currently shows numbers. Add a 7-day/30-day listening bar chart, a streak counter badge, and a "best listening day" highlight to make stats more engaging and motivating.

  • Bar chart uses data already tracked in play_history_service.dart
  • "Best day" bar highlighted in brighter purple with a "best" label
  • Streak counter uses consecutive days with >1 minute of listening
  • Toggle between 7-day and 30-day views
