Player Metadata Study

Make the title block feel editorial, not like three leftover captions.

For this screen, the biggest improvement is not a wilder cover treatment. It is turning the metadata under the cover into a clear hierarchy: small context first, one dominant title, then distinct credits for author and narrator.

Why the current stack feels weak

Subtitle should not compete with the title

If the subtitle sits as another centered line under the title, it looks like body copy. It works better as an overline or badge that frames the title before you read it.

Narrator deserves a different shape

Narrator is useful metadata, but it should not read like a faded third line. Put it in a structured credit row so it feels intentional and easier to scan.

Centered text alone gets mushy fast

Cover, title, subtitle, author, narrator, chapter, and time all stack into one long column. Breaking credits into cards reduces visual blur without making the screen busier.

Keep the cover as hero, not the only anchor

The cover still carries emotion. The metadata block should feel like a designed companion to it, not just labels dropped underneath.
