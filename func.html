Kitzi — Functionality Improvements

Feature & capability improvements grounded in the existing architecture

1

Bookmarks with Text Notes

Medium

The player has a bookmark concept in journal sheets but no way to attach a text note to a timestamp. Allow users to long-press the rewind button (or tap a dedicated bookmark icon) to drop a pin at the current position and optionally type a short note. Notes are stored locally and synced to ABS server if the API supports it.

Where it fits in the codebase: 
playback_journal_service.dart  →  add BookmarkEntry model with (bookId, positionMs, note, chapterId)
journal_sheets.dart            →  extend UI to show bookmarks tab with list + tap-to-seek
playback_repository.dart       →  expose addBookmark() / getBookmarks() streams
SQLite (existing)              →  new 'bookmarks' table (local persistence)
api_client.dart                →  POST to ABS /api/items/{id}/bookmarks when online
2

Multiple Server Support

Hard

Many Audiobookshelf users run multiple servers (home + work, personal + family). Currently switching servers requires logging out and back in, losing all cached data. Add a server switcher: save multiple server profiles, switch with one tap, and maintain separate download caches per server.

Where it fits in the codebase: 
auth_repository.dart     →  add ServerProfile model (url, username, token, libraryId)
                             store list of profiles; add activeProfile getter
api_client.dart          →  reads baseUrl + token from auth_repository.activeProfile
books_repository.dart    →  namespace SQLite tables by server ID (or use separate DB files)
download_storage.dart    →  separate download directories per server ID
settings_page.dart       →  new "Manage Servers" section at top
3

Skip Silence + Configurable Intro/Outro Skip

Medium

Long audiobooks often have seconds of silence between chapters, intros, and outros. Add two features: (a) auto-skip silence (detect <X dB threshold for >0.5s and skip), and (b) configurable chapter start/end skip timings (e.g. "skip first 5s and last 8s of every chapter"). Both toggle in settings.

Where it fits in the codebase: 
playback_speed_service.dart  →  add silence detection using just_audio's playerStateStream
smart_rewind_service.dart    →  new SilenceSkipService follows same pattern
playback_repository.dart     →  add introSkipSeconds / outroSkipSeconds prefs
settings_page.dart           →  two new numeric pickers under Playback section
ui_prefs.dart                →  persist introSkipMs, outroSkipMs, skipSilenceEnabled
4

Listening Goals & Daily Streak

Medium

Users have detailed listening history already tracked but no motivational goal system. Add a daily listening goal (e.g. 30 minutes/day), a streak counter for consecutive days meeting the goal, and a local notification reminder if the day's goal hasn't been reached by a configurable time.

Where it fits in the codebase: 
play_history_service.dart    →  todayListenedMinutes() — already has session data
ui_prefs.dart                →  add dailyGoalMinutes, reminderEnabled, reminderTime
stats_page.dart              →  add Goals card using circular progress indicator
notification_service.dart    →  schedule daily reminder notification via flutter_local_notifications
5

Chapter-Level Download Selection

Hard

Full book downloads are large. For users with limited storage, let them download only specific chapters. On the Book Detail page, show a chapter list with individual download buttons. The player detects which chapters are local vs. streamed and handles the mix transparently.

Where it fits in the codebase: 
downloads_repository.dart    →  add downloadChapter(bookId, chapterId) alongside downloadBook()
book_detail_page.dart        →  replace single download button with chapter-level list
playback_repository.dart     →  when seeking, check if chapter is local or needs streaming
download_storage.dart        →  store chapters keyed by (bookId, chapterId)
download_button.dart         →  extend to accept optional chapterId
6

Android Home Screen Widget

Hard

Power users want playback controls on their home screen. A 4×2 widget showing cover art, title, progress, and play/pause + skip buttons would reduce friction enormously — no need to open the app to resume an audiobook.

Where it fits in the codebase: 
android/app/src/main/        →  new HomeWidgetProvider.kt (AppWidgetProvider)
                                uses Android RemoteViews, communicates via SharedPreferences
MainActivity.kt              →  handle widget action intents (play/pause/skip)
audio_service_handler.dart   →  emit widget-update broadcasts on state change
pubspec.yaml                 →  add home_widget: ^0.6.0 Flutter plugin for widget bridge
7

Playback Queue — "Up Next" for Audiobooks

Medium

Currently the app plays one book and stops. Add a lightweight "Up Next" queue. Long-press a book cover to "Add to Queue", and when the current book finishes, the next one auto-starts. Queue is visible from the player with drag-to-reorder.

Where it fits in the codebase: 
playback_repository.dart     →  add playbackQueue: List<Book>, addToQueue(), onBookFinished()
audio_service_handler.dart   →  when track ends and queue non-empty, start next item
full_player_overlay.dart     →  add "Queue" tab alongside chapter list
books_page.dart              →  long-press context menu: "Play Next", "Add to Queue"
ui_prefs.dart                →  persist queue across app restarts
